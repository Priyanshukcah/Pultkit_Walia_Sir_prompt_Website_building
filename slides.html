Why the Mega Prompt works

Five rules, dialled to 11.

  • Consolidation maxed. What would be 15 prompts becomes one structured brief.
  • Numbers everywhere. Hex codes, px, ms, easings — different students get the same result.
  • DO + DO-NOT lists. Guardrails prevent the AI from "improving" things it shouldn't.
  • Demand deliverables. Files touched + test steps + the single biggest visual change.
  • Accessibility baked in. prefers-reduced-motion guard is part of the prompt.

Prompt 1 · Generate SQL with AI

Don't memorise SQL. Describe it.

prompt 1You are a Postgres expert helping a beginner.

I am using Supabase. Write a single CREATE TABLE
statement for a table called "form" that stores
contact-form submissions.

Columns:
1. id — bigint, auto-incrementing, primary key
2. created_at — timestamptz, default now()
3. full_name — text, not null
4. email — text, not null
5. subject — text, not null
6. message — text, not null
7. is_read — boolean, default false, not null

After the SQL, add a 3-line plain-English explanation
of what each column does and why we picked these types.
Strategy

  • Numbered columns + exact types → zero ambiguity.

Module 4 · Credentials

Two strings, one Amazon-login analogy.

🌐

SUPABASE_URL

The address of your project (like amazon.com).

🔑

SUPABASE_ANON_KEY

A public key that lets your front-end talk to the DB (like your login).

Find both in Supabase → Project Settings → API. We'll keep them in a separate file (supabase-config.js) — so we can .gitignore them before pushing to GitHub.

Prompt 2 · Wire the form

From fake-success to real-insert.

prompt 2 (key parts)1. Add Supabase JS CDN in contact.html <head>:
   <script src="https://cdn.jsdelivr.net/npm/
           @supabase/supabase-js@2"></script>

2. Create supabase-config.js:
   const SUPABASE_URL = "...";
   const SUPABASE_ANON_KEY = "...";
   const supabaseClient =
     supabase.createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

3. Link supabase-config.js BEFORE script.js
   (order matters!)

4. In script.js update the submit handler:
   - Read full_name, email, subject, message
   - supabaseClient.from('form').insert([{...}])
   - on error → red message
   - on success → existing green message + reset
   - log response for debugging

Append. Do not duplicate the handler.
Strategy
  • Explicit script order — the bug 80% of beginners hit.
  • Both success and error paths defined upfront.
  • Scope lock: append, don't duplicate.

Prompt 4 · Secrets first

.gitignore before git push. Always.

prompt 4Create a .gitignore in the project root that ignores:
- supabase-config.js   (contains my API keys)
- .DS_Store, Thumbs.db
- node_modules/
- .env, .env.local

Also create supabase-config.example.js — same shape,
empty strings — so other developers know what
variables to define.

Explain in 2 sentences why we never commit API keys
to GitHub, even for "anon" keys.
Why this prompt is mandatory
  • If you skip .gitignore, your DB credentials are on Google in 6 minutes.
  • The .example file teaches a real-world convention.
