Module 2

Project
Initialisation

Prompt 1 · Scaffold

Three files, one chat message.

paste into CursorYou are pair-programming with a complete beginner.

Create a starter web project in the current folder with these three files:
1. index.html  – HTML5 boilerplate titled "Aanya Sharma | UX Designer"
2. styles.css  – empty stylesheet
3. script.js   – console.log("Portfolio loaded ✅")

Inside index.html, link styles.css in the <head> and
script.js just before </body>.

After creating the files, explain in plain English what
each file does and why we link them in those places.
Why this prompt works
  • Persona for the AI ("beginner") forces simpler output.
  • Numbered deliverables → no ambiguity.
  • Exact placement of <link> and <script> teaches through the prompt.
  • "Explain in plain English" — Rule #5 in action.
Module 3

Sticky
Navigation

Prompt 2 · Navigation bar

Eight requirements, one message.

prompt 2In index.html, add a sticky nav at the top of <body>.

Requirements:
- Brand left: "Aanya Sharma"
- 4 links right, evenly spaced:
  Home, About, Projects, Contact
- Links → index.html, about.html,
         projects.html, contact.html
- Dark bg #111827, white text, 16px, 64px tall
- Sticky position
- Hover: text → #a78bfa, 0.2s transition
- Current page link has .active → purple underline

Put styles under /* === NAVIGATION === */ in styles.css
Add HTML comments above the nav explaining each part.
Strategy unpacked
  • Exact hex codes, exact px values (Rule 1).
  • Eight specs in one prompt (Rule 3 — consolidation).
  • Future-proofing: .active class declared before other pages exist.
  • Stylesheet hygiene from message #2: section headers.

Prompt 3 · Hero

Above-the-fold, fully specified.

prompt 3Below the nav in index.html, add a hero section.

Layout: full width, 80vh tall, centred content
Background: linear-gradient #4f46e5 → #9333ea (TL→BR)
White text, centred both axes

Content (in order):
1. Eyebrow "PORTFOLIO 2026" — 14px, uppercase,
   letter-spacing 4px, 60% opacity
2. h1 "I design products people actually enjoy using."
   56px, bold, max-width 800px
3. p sub "Senior UX Designer with 5 yrs in fintech,
   edtech and healthcare." — 20px, 80% opacity
4. CTA button "See My Work →" → projects.html
   white bg, #111827 text, 16/32 padding, 999px radius
   subtle shadow, lift 4px on hover (0.2s)

Style under /* === HERO === */
Briefly explain why a Hero matters for first-time visitors.
What we did
  • Visual hierarchy spelled out as a numbered list.
  • Gradient direction stated — no guesswork.
  • Every interactive state (hover, lift) specified up-front.
  • "Why a Hero matters" — converts a build into a lesson.
Prompt 4 · Iteration pattern

Anchor + delta + "don't touch anything else".

prompt 4 — refinementTwo refinements to the hero:

1. The headline feels too big on a 13" laptop.
   Reduce h1 to 48px and add a media query
   so it becomes 36px below 640px.

2. Add a subtle floating animation to the CTA —
   move up 3px and back over 2s, loop infinitely,
   CSS @keyframes.

Do not touch anything else.
The single most useful pattern in this course
  • Anchor the problem in user words.
  • Delta with exact numbers.
  • Lock the scope — "do not touch anything else" prevents the AI from "improving" other things.

Module 13 · Bonus

Debugging with AI — symptom + suspect + scope.

prompt 12 — debugThe layout of index.html is broken — the footer is
overlapping the hero. I have not changed any CSS
recently, only HTML. Please:

1. Read index.html carefully.
2. Find the structural mistake.
3. Fix only that mistake — do not refactor.
4. Tell me what was wrong and how I could have
   spotted it myself (DevTools, linter, etc.).
The bug-report pattern
  • Symptom — what's visibly wrong.
  • Suspect — what you recently changed.
  • Scope — fix only this.
  • Self-improvement — "how could I have spotted it?"
